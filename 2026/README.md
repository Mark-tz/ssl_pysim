# 2026 机器人专业考核 —— 分布式协作任务分配

## 场景背景

想象两台地面移动机器人在一个 **10m × 10m** 的场地中协同工作。
场地内随机分布着若干**任务点**，两台机器人需要合作，尽快完成对所有任务点的覆盖。

两台机器人的编号分别为 **A（编号0）** 和 **B（编号1）**，
它们的固定起始位置如下：

```
(0,0)────────────────────(10,0)
  │   A起点(1,1)             │
  │                          │
  │         场地              │
  │                          │
  │                 B起点    │
  │                 (9,1)    │
(0,10)──────────────────(10,10)  ← 目标区域在上方
```

### 关键规则

1. 两台机器人**独立决策**，没有集中调度——每台机器人只能看到对方的位置和当前目标，需要自己判断去哪里
2. 机器人到达任务点 **0.35m** 范围内，视为完成该任务
3. 如果两台机器人同时前往同一个任务点，**先到达的机器人得分**，后到达的白跑一趟，然后重新选择目标
4. 任务点会动态变化：完成后消失，场地适时会补充新的任务点

## 实验任务

你需要在 `strategy.py` 文件中完成 `choose_task` 函数。
这个函数负责告诉机器人：**"此刻，我应该去哪个任务点？"**

### 函数签名

```python
def choose_task(my_pos, my_id, partner_pos, partner_target, tasks):
    """
    参数：
        my_pos         : tuple (x, y)  —— 我的当前坐标（单位：米）
        my_id          : int (0 或 1)  —— 我的机器人编号
        partner_pos    : tuple (x, y)  —— 队友的当前坐标
        partner_target : int           —— 队友正在前往的任务点编号
                                          （-1 表示队友此时暂无目标）
        tasks          : list[(x, y)]  —— 当前所有未完成任务点的坐标列表

    返回：
        int —— 你希望前往的任务点编号（即 tasks 中的下标，0 ~ len(tasks)-1）
    """
```

### 调用时机

每当机器人**刚完成一个任务**（或仿真开始时还没有目标），此函数被调用一次。
函数每次返回一个整数，机器人就会朝那个任务点出发，直到到达（或任务被队友抢先完成）。

## 设计思路

### 第一步：理解问题

下面考虑一个简单情况：

- 剩余任务点：A(2,5)、B(8,5)
- 我在 (1,5)，队友在 (9,5)
- 我距 A 只有 1m，距 B 有 7m
- 队友距 A 有 8m，距 B 只有 1m

此时最优分配显然是：**我去A，队友去B**，各自负责离自己更近的那个。
这就避免了两人抢同一个任务点的浪费。

### 第二步：从参数中获取信息

你可以用 `my_pos`（我的位置）和 `tasks`（任务列表）计算"**我到各任务的距离**"：

```python
import numpy as np

my_pos = np.array(my_pos)
# 计算我到每个任务的距离
my_dists = [np.hypot(my_pos[0] - t[0], my_pos[1] - t[1]) for t in tasks]
```

你还可以用 `partner_pos` 计算"**队友到各任务的距离**"：

```python
partner_pos = np.array(partner_pos)
partner_dists = [np.hypot(partner_pos[0] - t[0], partner_pos[1] - t[1]) for t in tasks]
```

你也知道 `partner_target`（队友当前正在前往的任务点编号）。

### 第三步：设计你的策略

综合以上信息，尝试回答这个问题：

> **如何利用"我和队友各自到不同任务的距离"信息，让两台机器人尽量少冲突、多完成？**

## 常用 numpy 函数参考

### np.hypot —— 计算两点距离

```python
import numpy as np
dist = np.hypot(3.0, 4.0)   # dist = 5.0（勾股定理：√(3²+4²)）

# 计算从点 (1,2) 到 (4,6) 的距离：
dist = np.hypot(4-1, 6-2)   # dist = 5.0
```

### np.argmin —— 找最小值的下标

```python
import numpy as np
dists = [5.0, 1.2, 3.4, 0.8]
idx = np.argmin(dists)   # idx = 3（dists[3]=0.8 最小）
```

### np.array —— 创建数组，支持向量运算

```python
import numpy as np
a = np.array([1.0, 2.0, 3.0])
b = np.array([4.0, 5.0, 6.0])
c = b - a      # c = [3.0, 3.0, 3.0]
d = a * 2      # d = [2.0, 4.0, 6.0]
```

### np.where —— 按条件筛选下标

```python
import numpy as np
values = np.array([1.0, -2.0, 3.0, -1.0])
# 找出大于0的元素的下标：
indices = np.where(values > 0)[0]   # indices = [0, 2]
```

## 仿真界面说明

运行 `main.exe`（或 `python main.py`）后，界面如下：

```
┌───────────────────────┬──────────────────────┐
│                       │   完成进度折线图      │
│   主仿真视图           │  （任务数 vs 时间）   │
│                       ├──────────────────────┤
│  蓝圈 = 机器人A        │   当前状态文字面板    │
│  红圈 = 机器人B        │                      │
│  灰点 = 待完成任务     │  Run X / 10          │
│  虚线 = 当前规划路径   │  时间: xx.x / 30 s   │
│                       │  A完成: xx            │
│                       │  B完成: xx            │
│                       │                      │
└───────────────────────┴──────────────────────┘
```

- 按**空格键**可以暂停 / 继续仿真
- 调整 `strategy.py` 中的 `SIM_PAUSE_TIME` 可以控制动画速度：
  - `0.01`：快速（每5步刷新一次画面，推荐调试阶段使用）
  - `0.05`：中速（每步都刷新，方便观察细节）
  - `-1`：手动单步（每按一次空格前进一帧）
- **极速模式**：将 `strategy.py` 中的 `EXAM_MODE` 设为 `True`，将完全关闭可视化界面，仅输出最终结果，适合快速跑分测试。

- 确认策略无误后，开启 `EXAM_MODE = True` 快速运行 10 轮，看总分是否满意

## 评分标准

正式测试时，将由现场随机公布一个 **SEED**（随机种子），所有人使用同一个 SEED 运行，确保地图环境一致。

| 优先级 | 指标 | 说明 |
|:---:|:---|:---|
| 第 1 | **10轮合计完成任务数**（总分） | 越高越好 |
| 第 2 | 最高单轮得分 | 总分相同时，比较单次最优表现 |
| 第 3 | 最低单轮得分 | 仍相同时，比较稳定性 |

每轮仿真时长为 **30秒**，结束后自动进入下一轮，共测试 **10轮**。

## 参考说明：什么是"分布式决策"？

想象两个人在超市里分头购物，每人手里有一份购物清单，但两份清单里有重叠的商品。如果两人各自只想着"我去找最近的商品"，很可能在同一个货架前相遇，互相等待。

更好的方式是：每人观察对方在哪里，**主动去对方不擅长、不方便覆盖的区域**，这样两人就能高效分工，总体完成时间大幅缩短。

这个题目考察的正是这种"**在没有集中调度的情况下，如何通过信息共享实现自发分工**"的逻辑。
