# 2026 考核题目 —— 助教操作手册

> 本文档供**出题人 / 助教**使用，包含工程原理、考前准备、现场流程和赛后评分的完整说明。
> 学生不需要阅读此文件。


## 二、工程原理

### 2.1 整体架构

```
┌──────────────────────────────────────────────────────────────┐
│                         main.exe                             │
│                                                              │
│  ┌─────────────┐   调用    ┌──────────────────────────────┐  │
│  │  仿真引擎    │ ────────► │  strategy.py（学生编写）      │  │
│  │  Robot      │           │  choose_task(my_pos,         │  │
│  │  TaskMgr    │ ◄──────── │               partner_pos,   │  │
│  │  Visualizer │  返回idx  │               ...)            │  │
│  └─────────────┘           └──────────────────────────────┘  │
│         │                                                     │
│         ▼                                                     │
│  ┌─────────────┐                                             │
│  │ save_result │ ──► res2026_<学号>.gzip                     │
│  │ (HMAC签名)  │                                             │
│  └─────────────┘                                             │
└──────────────────────────────────────────────────────────────┘
```

`main.exe` 在启动时会把自身所在目录加入 Python 模块搜索路径，
然后 `import strategy`，因此只要 `strategy.py` 与 `main.exe` **放在同一目录下**，
学生对 `strategy.py` 的任何修改都会立即生效，无需重新编译。

### 2.2 仿真逻辑概述

| 参数 | 值 | 说明 |
|---|---|---|
| 场地 | 10×10 m | 固定 |
| 机器人A起点 | (1.0, 1.0) | 固定 |
| 机器人B起点 | (9.0, 1.0) | 固定 |
| 机器人速度 | 2.0 m/s | 固定 |
| 任务完成半径 | 0.35 m | 固定 |
| 单轮时长 | 30 s | 固定 |
| 测试轮数 | 10 轮 | 固定 |
| 初始任务数 | 10 | 每轮重置 |
| 任务刷新 | 剩余≤3时补5个 | 动态 |
| 最大任务生成数 | 60 / 轮 | 固定 |

每轮的**随机种子**由 `strategy.py` 中的 `SEED` 变量控制：
- 学生调试时可以设任意值（或 `None` 让每次随机）
- 正式考试时统一设为当场公布的种子数字

### 2.3 结果文件格式与防篡改

结果文件（`res2026_<学号>.gzip`）结构：

```
[ 32 字节 HMAC-SHA256 签名 ][ gzip 压缩的 pickle 数据 ]
```

pickle 数据解包后为 Python 字典：

```python
{
    'student_id': '3250101234',   # 学号
    'seed':       9527,           # 使用的随机种子
    'runs':       [52, 48, ...],  # 10 轮各轮得分（list[int]）
    'total':      502,            # 10 轮总分
    'ts':         1741234567,     # Unix 时间戳（生成时间）
}
```

签名使用的密钥（`_HMAC_KEY`）硬编码在 `main.py` / `main.exe` 内部。
学生拿到的只是 `main.exe`（已打包，源码不可见），因此**无法伪造签名**。
`read_res.py` 会在读取每份文件时自动验证签名，失败的文件会被标记为 `[无效]` 并排除在排名之外。

## 三、考前准备

### 3.1 打包 main.exe（Windows 环境）

```bash
# 安装依赖
pip install numpy matplotlib pyinstaller

# 进入题目目录
cd 2026/

# 运行打包脚本
build.bat
```

打包完成后，`dist/main.exe` 即为发放给学生的可执行文件。

> **注意**：PyInstaller 打包具有平台依赖性，**在 Windows 上打包才能生成 Windows 的 .exe**。
> 若考试在 Linux/macOS 环境下进行，请改用 `build.sh`，生成的文件名为 `main`（无后缀）。

### 3.2 验证打包结果

打包完成后，将以下文件放在同一目录，手动测试一次：

```
test_dir/
├── main.exe
├── strategy.py   ← 使用默认策略（STUDENT_ID 改为测试学号）
└── README.md
```

双击 `main.exe`，确认：
1. 弹出仿真窗口，能正常看到两个机器人移动
2. 10 轮结束后控制台打印总分
3. 生成 `res2026_<测试学号>.gzip` 文件

### 3.3 准备发放包

```
发布包/
├── main.exe      ← dist/main.exe（打包产物）
├── strategy.py   ← 使用原始版本（STUDENT_ID = "请填写你的学号"，SEED = None）
└── README.md
```

> **不要**把 `read_res.py`、`build.bat`、`README_TA.md` 等文件发给学生。

## 四、考试现场流程

### 步骤 1：分发题目包

将 `发布包/` 中的三个文件复制给每位学生（U盘 / 网络共享 / 邮件均可）。

告知学生：
- 使用 VSCode 打开 `strategy.py`
- 将 `STUDENT_ID` 改为自己的学号
- 在 `choose_task` 函数中编写策略
- 双击 `main.exe` 调试，可反复修改

### 步骤 2：宣布随机种子

在截止时间前约 **5 分钟**，当场生成随机种子并宣布（建议用投影仪展示）：

```python
# 可以用这个命令现场生成
python -c "import random; print(random.randint(1000, 9999))"
```

宣布后，所有学生：
1. 将 `strategy.py` 中的 `SEED = None` 改为公布的数字（例如 `SEED = 7391`）
2. 运行 `main.exe`，等待 10 轮测试自动完成
3. 将生成的 `res2026_<学号>.gzip` 文件提交（按 `学号_姓名_res2026.gzip` 重命名）

> **时间参考**：10 轮仿真（`SIM_PAUSE_TIME = 0.01`）约需 **2~4 分钟**完成。
> 建议在宣布种子后给学生 **8 分钟**的提交窗口。

### 步骤 3：收集结果文件

收集所有学生提交的 `.gzip` 文件，放入同一目录，例如：

```
submissions/
├── 3250101001_张三_res2026.gzip
├── 3250101002_李四_res2026.gzip
├── 3250101003_王五_res2026.gzip
└── ...
```

### 5.2 排名规则

| 优先级 | 指标 | 说明 |
|:---:|---|---|
| 第1 | **10轮总分**（合计完成任务数） | 主要排名依据 |
| 第2 | 最高单轮得分 | 总分相同时比较单次峰值 |
| 第3 | 最低单轮得分 | 仍相同时比较稳定性（最低分越高越好） |

### 5.3 处理异常情况

**情况1：文件显示 `[无效] HMAC 签名不匹配`**

学生可能修改了结果文件内容。该文件将被排除在排名之外。
建议向学生说明原因，必要时要求重新测试。

**情况2：学生使用了错误的 SEED**

评分脚本会打印每份文件使用的 `SEED`，若某学生 `SEED` 与其他人不同，
说明该学生未正确填写种子，其结果不与其他人在相同地图条件下产生，
需要酌情处理（建议要求重测）。

**情况3：`total` 与 `runs` 总和不一致**

脚本会自动检测此类数据内部矛盾，并将文件标记为 `[无效]`。

## 六、评分参考基准

以下为典型策略在正常条件下的得分区间（仅供参考，实际因地图随机性有波动）：

| 策略水平 | 典型总分 | 说明 |
|---|---|---|
| 不作答（默认贪心） | 380 ~ 430 | 两机器人频繁抢同一任务 |
| Level 2：简单避让 | 450 ~ 500 | 避开队友当前目标，但有隐式冲突 |
| Level 3：Voronoi分区 | 520 ~ 560 | 选"我比队友更近"的任务，近似最优 |
| Level 4+：动态权衡 | 550 ~ 600 | 进一步考虑距离优势与绝对距离的平衡 |

> **考察的核心洞察**：
> 学生需要通过观察仿真（"两台机器人总是扑向同一个任务"）发现冲突问题，
> 并推导出"**选择我比队友距离更近的那些任务中，离我最近的那个**"这一
> 隐式 Voronoi 划分策略。这一思路大模型难以直接给出，需要学生自己从仿真中观察并推理。

## 七、常见问题

**Q：`main.exe` 打开后一片黑，闪退了怎么办？**

A：通常是缺少 matplotlib 后端。尝试在同目录下创建 `matplotlibrc` 文件，内容为 `backend: TkAgg`。

**Q：学生反映"ImportError: No module named strategy"？**

A：确认 `strategy.py` 与 `main.exe` 在**同一目录下**，且文件名完全为 `strategy.py`（注意大小写）。

**Q：可以用 Python 直接运行而不用 exe 吗？**

A：可以。学生在安装了 numpy 和 matplotlib 的环境中，在 `2026/` 目录下执行：
```bash
pip install numpy matplotlib
python main.py
```
效果与 exe 完全相同。

**Q：如何修改题目难度？**

A：修改 `main.py` 开头的常量：
- 增加 `MAX_TOTAL_TASKS` 或减少 `SIM_TIME` → 任务更难完完
- 调整 `REFRESH_TRIGGER` / `REFRESH_BATCH` → 改变任务密度
- 修改 `ROBOT_SPEED` → 影响整体节奏

修改后需重新打包。

**Q：`read_res.py` 可以在 Windows 上运行吗？**

A：可以。只需有 Python 环境（无需 numpy/matplotlib），直接 `python read_res.py <目录>` 即可。
